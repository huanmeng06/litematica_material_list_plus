# LMLP v1.8.0

本次更新是一轮面向大型整合包、材料拆分正确性和跨版本视觉一致性的完整升级，并新增 Minecraft 1.21.10、1.21.11 与 26.1.2 支持。

## ⚠️ 版本支持变更

LMLP v1.8.0 支持以下 Minecraft Fabric 版本：

| Minecraft | Release 资产 |
| --- | --- |
| 1.20.1 | `litematica-material-list-plus-1.8.0+mc1.20.1.jar` |
| 1.20.4 | `litematica-material-list-plus-1.8.0+mc1.20.4.jar` |
| 1.20.6 | `litematica-material-list-plus-1.8.0+mc1.20.6.jar` |
| 1.21.1 | `litematica-material-list-plus-1.8.0+mc1.21.1.jar` |
| 1.21.10 | `litematica-material-list-plus-1.8.0+mc1.21.10.jar` |
| 1.21.11 | `litematica-material-list-plus-1.8.0+mc1.21.11.jar` |
| 26.1.2 | `litematica-material-list-plus-1.8.0+mc26.1.2.jar` |

> **Minecraft 1.21.4 和 1.21.5 从 v1.8.0 起不再支持，也不会提供对应 JAR。**
> 这两个 Minecraft 版本的最后可用版本为 LMLP v1.7.0。请勿将其他 Minecraft 版本的 v1.8.0 JAR 混用到 1.21.4 或 1.21.5。

## 🚀 性能优化

- 最小子材料改为分段计算与后台预热，降低大型材料列表首次进入时的主线程停顿。
- 加入计算预算、配方查询缓存、结果签名与缓存失效机制，减少重复解析。
- 循环配方会提前终止，避免递归放大和大型整合包中的卡死。
- 优化首次切换最小子材料页面、渲染层页面和大量配方候选时的响应速度。
- 保持材料总数、缺少数、已有数和最终展示结果不变。

## 🧱 材料与配方修复

- 修复多产出配方、重复槽位、合成次数和材料数量的缩放错误。
- 修复煤炭块、白色床、骨粉/骨块及部分可逆压缩材料被错误继续拆分的问题。
- 限制“任意原木”等选择组的递归范围，避免把木材候选错误套用到无关材料。
- 从当前物品注册表和标签动态识别新原木，使模组新增木材进入“任意原木”候选面板。
- 修复配方 Tooltip、配方跳转、快速合成和“移动到容器中”按钮的兼容问题。

## 🔴 原点标记与渲染

- 各受支持版本统一使用对应 Minecraft 原版信标光束的纹理、几何和动画。
- 保持红色 OMMC 风格光束、靶心标记、红框黑底坐标标签和仅注视时显示标签的行为。
- 光束可以穿过实体方块，并显示在 Litematica 蓝图、云层、天气和粒子之上。
- 修复光束消失、被蓝图覆盖、标签缺失、箭头异常和渲染状态污染。

## 🖥 界面与交互

- 修复绑定热键后重置按钮不能立即启用、必须先点击搜索框的问题。
- 统一禁用物品图标效果：保留物品模型并覆盖中性灰遮罩，移除高开销且容易闪烁的离屏 framebuffer 灰度转换。
- 修复列表超行渲染、面板切换失败、Tooltip 层级和部分按钮显示问题。
- 保持 1.20.6 开发版既有操作、效果和 UI 行为，并迁移到所有受支持版本。

## 📦 安装

1. 从 Assets 下载与 Minecraft 版本完全对应的 JAR。
2. 删除 `mods` 文件夹中的旧版 LMLP JAR，确保只保留一个版本。
3. 安装与目标 Minecraft 版本匹配的 Fabric API、Litematica、MaLiLib 和 JEI。
4. v1.8.0 使用 JEI 集成，不再使用旧版 REI 集成。
