# LMLP 开发日志

这个 README 只记录 `dev-newFeature` 分支的开发过程和每个小版本做了什么。完整的项目介绍、安装说明、功能说明和截图展示放在 `main` 分支维护。

当前正式版：`v1.6.54`

当前构建：`1.6.54+mc1.20.6`

适配目标：Minecraft `1.20.6` / Fabric / Litematica / MaLiLib / REI

## 版本说明

### v1.6.52

- 给"我找不到投影"按钮加上距离判断：这个按钮之前只要判断玩家和缓存记录处在同一个维度就会一直显示，跟玩家实际有没有去过现场完全没关系——语义上这个按钮应该表示"你已经到现场看过、确实没找到"，但触发条件只看维度不看距离，导致玩家在这个维度的任何位置都会一直挂着这个按钮。现在复用了"高亮原点"功能里已有的坐标解析和距离计算（`PlacementOriginMarker.parseOrigin` + 到玩家的三维距离），只有玩家离这条记录的原点坐标 64 格以内时才会显示这个按钮；离得远的话这一行不显示任何操作按钮（这不是新状态，玩家在不同维度时今天就是这个表现，现在只是把同一种表现的适用范围从"维度不对"扩展到"维度对但距离不够近"）。
- 同步 `fabric.mod.json` 和运行时 `MOD_VERSION` 到 `1.6.52+mc1.20.6`。
- 构建产物改为 `1.6.52+mc1.20.6`。

### v1.6.51

- 修复 v1.6.50 里配方悬浮面板"配方 1: ... / 缺少 合成 X"这一行的问题：放不下时直接把"缺少"那部分整体丢弃、不留任何提示（"确实被隐藏了，但是没有..."），用户反馈更希望像别处一样把放不下的信息换行显示出来，而不是默默丢掉。现在这一行放不下时改为换成两行：第一行是"配方 1: ..."，第二行是带颜色的"缺少 合成 X"（颜色代码作为整体换到下一行，不会被从中间切断导致乱码）。为此把面板高度的计算（`getHeight`/`getTargetHeight`/`getOuterHeight`/`getTargetOuterHeight`）都改成了按可用宽度感知，换行时多出的一行高度会正确预留出来，悬浮检测和点击展开/收起的坐标计算也跟着同步更新，不会因为多出一行而错位。

### v1.6.50

- 修复配方悬浮面板（点击物品旁的箭头展开的"配方 1: ... / 缺少 合成 X"面板）里的文字不随窗口/行宽变窄而收缩：面板本身有裁剪（scissor），行宽变窄时文字会被硬生生切掉一半，很难看（比如"合成 872"被从中间截断）。现在面板里所有单行文字（物品名称、配方行、"仍需子材料"标签、每个材料行的"名称: 数量"、还有更多提示）都按可用宽度用 `...` 截断：
  - 材料行的数量部分（红/绿色的缺少数量）永远完整显示，只收缩前面的名称，保证"到底还缺多少"这个最关心的数字不会被切；
  - 配方行（"配方 1: ... / 缺少 合成 X"）末尾的"缺少"部分带颜色代码，直接从中间截断可能会截断颜色代码本身导致乱码，所以采用整体降级：放得下就完整显示，放不下就只显示前半句（不含颜色后缀），还是放不下才对前半句做省略号截断——不会出现颜色代码被腰斩的情况。
