# LMLP 开发日志

这个 README 只记录 `dev-newFeature` 分支的开发过程和每个小版本做了什么。完整的项目介绍、安装说明、功能说明和截图展示放在 `main` 分支维护。

当前正式版：`v1.4.18`

当前构建：`1.4.18+mc1.20.6`

适配目标：Minecraft `1.20.6` / Fabric / Litematica / MaLiLib / REI

## 版本说明

### v1.4.18

- 在 `最小子材料` 视图中隐藏每行右侧的原版“忽略”按钮，并禁止该视图中的行点击继续触发忽略按钮逻辑。
- 优化任意材料命名：只有候选材料超过一个时才显示 `任意` 前缀，单个石英物品会显示具体物品名；多候选石英类材料改为显示 `任意石英类物品`，减少和石英块/下界石英物品之间的歧义。

### v1.4.17

- 修复首次进入 `最小子材料` 和点击“刷新”后列宽不一致的问题：原版 `GuiMaterialList` 会在构造和材料列表任务完成时用 `getMaterialsAll()` 调用一次全量列宽计算，这会把主材料清单宽度写入 LMLP 的列宽状态；现在拦截这两次全量 sizing，只保留当前列表 `listContents` 刷新完成后的页面级列宽计算。
- 列宽计算数据源因此固定为当前页面实际显示行，避免刷新前后因为原版全量 sizing 调用顺序不同而出现不同列宽。

### v1.4.16

- 继续修复 `最小子材料` 视图刷新后的数量列贴靠/重叠问题：加大数量列测量安全余量，并把 `总计 -> 缺少 -> 可用` 的列间距改为统一常量，确保列表总宽度和实际列位置使用同一套间距。
- `项目 -> 总计` 的间距也同步改为布局常量，避免后续调整列宽时出现外框宽度和行内文字位置不一致。

### v1.4.15

- 修复材料列表列宽计算时机：现在会在当前页面的 `listContents` 完成刷新、重建 widgets 之前，按实际显示的材料行计算 `项目 / 总计 / 缺少 / 可用` 列宽，避免首次进入页面时先使用旧页面列宽。
- 修复 `最小子材料` 视图点击 Litematica 内置“刷新”后列宽可能按主材料清单或空列表状态回落的问题；刷新前后都使用当前页面实际行作为列宽数据源。
- 优化 `最小子材料` 视图加载体验：切换页面或刷新时不再主动清空已有最小子材料缓存；数据签名变化时会继续显示旧缓存，后台分帧构建新结果，完成后一次性替换，减少进入页面时的空白闪烁。
- 页面切换时列宽立即按切换后的页面落位；同一页面内因忽略长材料或刷新导致列宽变窄时仍保留平滑动画，列宽变宽时继续立即扩宽以避免文字重叠。

### v1.4.14

- 恢复材料列表列宽缩窄动画：忽略长材料或长数量后，列宽会平滑收回；如果刷新后需要更宽，则立即扩宽，避免动画过程中总计、缺少和可用列互相重叠。
- 优化 `最小子材料` 视图进入体验：递归汇总仍然分帧计算，但不再把中间结果逐行发布到列表，完成后一次性展示完整材料看板。
- 给数量列宽估算增加安全余量，降低中文数量格式在刷新后贴边或重叠的风险。

### v1.4.13
