# LMLP 开发日志

这个 README 只记录 `dev-newFeature` 分支的开发过程和每个小版本做了什么。完整的项目介绍、安装说明、功能说明和截图展示放在 `main` 分支维护。

当前正式版：`v1.4.44`

当前构建：`1.4.44+mc1.20.6`

适配目标：Minecraft `1.20.6` / Fabric / Litematica / MaLiLib / REI

## 版本说明

### v1.4.44

- `最小子材料` 来源展开面板标题改为 `物品名需求来源`，例如 `圆石需求来源`，让标题更短更直接。

### v1.4.43

- 移除悬停浮窗的三态模式选项，改为布尔开关 `关闭 Litematica 原版浮窗`。
- 新开关默认开启；开启时显示 LMLP 简略/详细浮窗，关闭时显示 Litematica 原版浮窗。
- 兼容迁移旧的 `enableLmlpHoverTooltip` 和 `hoverTooltipMode` 配置值。

### v1.4.42

- 修复切换游戏语言后，投影材料列表里的材料名称仍显示为上一个语言的问题。
- `最小子材料` 视图的材料缓存现在会把当前语言纳入签名，语言变化后会自动重建显示名称。

### v1.4.41

- `最小子材料` 来源展开面板删除第二行数量说明，来源列表直接接在首行说明下方。
- 首行中的 `数量` 单独使用与数量值一致的金色显示，使“主材料以及数量”的含义更醒目。

### v1.4.40

- `最小子材料` 来源展开面板的说明文案改为带具体物品名：`需要**物品名**的主材料` 和 `需要**物品名**的数量`，物品名使用加粗样式。
- 当某个最小子材料本身就是主材料时，来源面板不再显示“需要”说明和来源列表，改为显示 `**物品名**即为主材料`。

### v1.4.39

- `最小子材料` 来源展开面板不再折叠来源主材料，普通展开会直接按所需数量从高到低显示全部来源。
- 保留 Shift+左键打开独立来源详情页的入口，用于在更宽页面里查看同一批来源。

### v1.4.38

- 将 MaLiLib 列表滚动从按条目跳转改为像素级连续滚动，材料列表、最小子材料列表以及相关列表页面不再一格一格卡着走。
- 配方详情页和来源主材料详情页的滚轮输入改为像素累计，触控板或高精度滚轮的小幅滚动也会被保留，滚动手感更接近无级滑动。

### v1.4.37

- `最小子材料` 视图中 Shift+左键不再把内嵌来源面板拉长，而是打开一个独立的来源详情页。
- 来源详情页复用配方详情页的整体布局：顶部目标子材料信息、返回按钮、黑色遮罩背景、滚动内容区和描边列表框；下方内容改为列出所有需要该子材料的来源主材料。

### v1.4.36

- `最小子材料` 来源展开面板支持 Shift+左键显示完整来源列表；普通左键仍只显示按所需数量排序后的前 4 个来源。
- 折叠提示文案改为提示 `Shift 点击显示全部`，并缩短来源标题，避免标题行文字挤在一起。

### v1.4.35

- `最小子材料` 来源展开面板现在按每个来源主材料所需的当前子材料数量从高到低排序，只显示前 4 个贡献最大的主物品。
- 超过 4 个来源时，底部提示会明确说明其余较少需求来源已折叠，避免误以为还有额外可点击的二级展开入口。

### v1.4.34

- 调整紫珀块/紫珀柱这类配方候选的名称显示：配方摘要和 `最小子材料` 视图中统一显示为 `任意紫珀方块`，候选浮窗中继续列出具体可用的紫珀块、紫珀柱。
- 避免多候选行直接显示 `任一：紫珀块 / 紫珀柱` 这类配方描述文本，使行名和展开面板标题更像材料分类名称。

### v1.4.33

- 限制 `最小子材料` 来源展开面板的可见来源行数，红石粉等被大量主材料使用的物品不再把列表条目撑出一整屏空白。
- 来源超过可见行数时，面板底部会显示剩余来源数量提示，保留展开用途的同时避免页面无法完整显示。

### v1.4.32

- `最小子材料` 视图中，只要某一行代表多个可替换候选物品，行名都会使用黄色、加粗、下划线高亮；同木种的橡木原木/去皮橡木原木/橡木/去皮橡木等可变代表名也会和 `任意...` 行一样醒目。

### v1.4.31

- `recipeStopItems` 支持 `{color}` 占位符，一行 `minecraft:{color}_wool` 即可匹配 16 色羊毛，同样适用于染料、地毯、陶瓦、玻璃、混凝土等使用 Minecraft 颜色前缀的物品。
- 默认停止拆分列表改用 `{color}` 压缩 16 色染料、羊毛、地毯和陶瓦，配置界面不再展开出大量重复颜色行；加载旧配置时会把完整 16 色序列自动折叠成对应模式行。

### v1.4.30

- 将 16 种羊毛加入默认 `recipeStopItems` 停止拆分列表，避免最小子材料视图和递归拆分继续把羊毛拆成线或染料相关来源。

### v1.4.29

- 修复 `最小子材料` 视图里同一木种材料可能同时显示木板和原木/菌柄的问题：当配方候选已按来源木种收窄成单个木板时，会继续递归拆分该木板，而不是把它作为叶子材料保留。
- 真正跨木种的 `任意木板` 候选仍会停在候选行，不会被任意选择某一种木材继续拆分。

### v1.4.28

- 将 16 色陶瓦加入默认 `recipeStopItems` 停止拆分列表，避免白色陶瓦、橙色陶瓦等染色陶瓦在最小子材料视图和递归拆分中继续拆成普通陶瓦与染料。

### v1.4.27

- 继续清理 `任意木板` 来源残留：最小子材料汇总时，如果来源主材料本身带有明确木种，会按来源木种过滤木板/原木候选，白桦告示牌、绯红木栅栏门等特定木种来源不再落入全局 `任意木板`。
- 配方摘要中的同木种多木板候选也会显示为对应木板代表名，和同木种原木候选的收窄逻辑保持一致。

### v1.4.26

- 继续修正木质材料的任意分组：跨木种候选才会显示为 `任意原木`，同一木种内部的多个候选（例如白桦原木、去皮白桦原木、白桦木、去皮白桦木）会归到对应木种代表材料，不再混入全局 `任意原木`。
- REI 配方摘要中的同木种原木候选名称也同步收窄，避免白桦告示牌、绯红木栅栏门等特定木种来源继续显示在 `任意原木` 来源列表里。

### v1.4.25

- `最小子材料` 因此不再把需要特定木板/原木的主材料错误归入 `任意木板` / `任意原木`；真正无木种限制的配方仍会保留任意材料行。

### v1.4.24

- 最小子材料汇总阶段现在会记录每个叶子材料的来源主材料贡献，用于展开面板展示；不改变原有总计/缺少数量计算结果。

- 新增最小子材料候选浮窗访问接口，让列表级渲染时机调用具体条目的浮窗绘制逻辑，避免被行渲染顺序或列表裁剪覆盖。
- `任意...` 候选解释浮窗现在作为 `最小子材料` 页面专属说明直接显示，不再受普通材料悬浮提示模式影响。

### v1.4.17

- 修复首次进入 `最小子材料` 和点击“刷新”后列宽不一致的问题：原版 `GuiMaterialList` 会在构造和材料列表任务完成时用 `getMaterialsAll()` 调用一次全量列宽计算，这会把主材料清单宽度写入 LMLP 的列宽状态；现在拦截这两次全量 sizing，只保留当前列表 `listContents` 刷新完成后的页面级列宽计算。
- 列宽计算数据源因此固定为当前页面实际显示行，避免刷新前后因为原版全量 sizing 调用顺序不同而出现不同列宽。

### v1.4.16

- 继续修复 `最小子材料` 视图刷新后的数量列贴靠/重叠问题：加大数量列测量安全余量，并把 `总计 -> 缺少 -> 可用` 的列间距改为统一常量，确保列表总宽度和实际列位置使用同一套间距。
- `项目 -> 总计` 的间距也同步改为布局常量，避免后续调整列宽时出现外框宽度和行内文字位置不一致。

### v1.4.15

- 修复材料列表列宽计算时机：现在会在当前页面的 `listContents` 完成刷新、重建 widgets 之前，按实际显示的材料行计算 `项目 / 总计 / 缺少 / 可用` 列宽，避免首次进入页面时先使用旧页面列宽。
- 修复 `最小子材料` 视图点击 Litematica 内置“刷新”后列宽可能按主材料清单或空列表状态回落的问题；刷新前后都使用当前页面实际行作为列宽数据源。
- 优化 `最小子材料` 视图加载体验：切换页面或刷新时不再主动清空已有最小子材料缓存；数据签名变化时会继续显示旧缓存，后台分帧构建新结果，完成后一次性替换，减少进入页面时的空白闪烁。
- 页面切换时列宽立即按切换后的页面落位；同一页面内因忽略长材料或刷新导致列宽变窄时仍保留平滑动画，列宽变宽时继续立即扩宽以避免文字重叠。
